ℹ️ What is "seed_ward"?
Seeding creates a new run that starts from an existing stage's NDM checkpoint.
Instead of re-running from scratch, the flow resumes from the seeded stage.
Example: seed from floorplan → logic_opto starts directly with that floorplan.
Use this when: changing hooks/scripts without re-running earlier stages.

        📡 Teams Phone Alerts — Setup Guide
        ▼
      

    

    

      

        Set up Microsoft Teams alerts from the Copilot CLI in 5 minutes.
        After setup, the dashboard can send alerts to your phone for failures, stage completions, and more.
      


      
      

        
Step 1 — Add M365 Graph MCP to ~/.copilot/mcp-config.json

        
Inside mcpServers, alongside existing servers:

        
"m365-graph": {
  "type": "stdio",
  "command": "/nfs/site/disks/xeg.jgsfe_compute.0001/yrevah1/GITHUB/applications.manufacturing.intel.ai.agents/m365GraphAgent/m365GraphMCPServer/bin/Release/net8.0/linux-x64/publish/m365-graph-mcp",
  "args": [],
  "env": {},
  "cwd": "/nfs/site/disks/xeg.jgsfe_compute.0001/yrevah1/GITHUB/applications.manufacturing.intel.ai.agents/m365GraphAgent/m365GraphMCPServer/bin/Release/net8.0/linux-x64/publish"
}

        
      


      
      

        
Step 2 — Restart Copilot CLI

        
On first use of any Teams/Email tool, a browser window will open for Microsoft auth. Requires VNC display.

      


      
      

        
Step 3 — Install MSAL & Authenticate

        
python3 -m pip install msal
cp /nfs/site/disks/xeg.jgsbe_compute.0002/sbehar/scripts/scripts/teams_notify.py ~/scripts/
python3 ~/scripts/teams_notify.py --auth

        
Enter the device code at https://login.microsoft.com/device with your Intel account.

      


      
      

        
Step 4 — Create "Copilot CLI" self-chat group

        
python3 - << 'EOF'
import os, requests, msal
CLIENT_ID = "14d82eec-204b-4c2f-b7e8-296a70dab67e"
TENANT_ID = "46c98d88-e344-4ed4-8496-4ed7712e255d"
SCOPES    = ["Chat.ReadWrite","ChatMessage.Send","User.Read"]
CACHE     = os.path.expanduser("~/.local/share/M365GraphAgent/py_token_cache.json")
cache = msal.SerializableTokenCache()
cache.deserialize(open(CACHE).read())
app = msal.PublicClientApplication(CLIENT_ID,
    authority=f"https://login.microsoftonline.com/{TENANT_ID}", token_cache=cache)
token = app.acquire_token_silent(SCOPES, account=app.get_accounts()[0])["access_token"]
h = {"Authorization": f"Bearer {token}", "Content-Type": "application/json"}
me = requests.get("https://graph.microsoft.com/v1.0/me", headers=h).json()["id"]
r = requests.post("https://graph.microsoft.com/v1.0/chats", headers=h, json={
    "chatType": "group", "topic": "Copilot CLI",
    "members": [{"@odata.type": "#microsoft.graph.aadUserConversationMember",
                 "roles": ["owner"],
                 "user@odata.bind": f"https://graph.microsoft.com/v1.0/users/{me}"}]
}).json()
print(f"Chat ID: {r['id']}")
EOF

        
Copy the printed Chat ID.

      


      
      

        
Step 5 — Update SELF_CHAT in teams_notify.py

        
SELF_CHAT = "19:YOUR_CHAT_ID@thread.v2"  # ← paste ID from Step 4

      


      
      

        
Step 6 — Test it

        
python3 ~/scripts/teams_notify.py "Hello from CLI! 🚀"

        
✅ You should receive a message in your "Copilot CLI" Teams chat.
